# 0009: Proposals Are Probes

**Types:** ["recognition", "process"]

---

A proposal is not just a solution to be evaluated. It is an instrument that discovers structure in the problem space.

1. **Checking reveals new problems.** When you evaluate a proposal against the problem map, three kinds of discovery happen: proposal-originated surfaces (the proposal addresses something the map missed), emergent detail (the proposal's treatment reveals depth the map didn't capture), and coverage gaps (the proposal has no treatment for a known axis). All three feed back into the problem definition — which grows.

2. **A "wrong" proposal teaches more than an "easy" one.** If the proposal is confidently wrong about an axis, it teaches you about that axis. If it's right, you learn nothing new. The probe that misses teaches more than the probe that hits.

3. **Solo bounce works too.** Think-question-think-question loop against yourself. The bounce doesn't require two — but two enhances it. Solo bounce has a threshold: relevance gate. If the next question doesn't pass the threshold, the loop terminates.

- The probe needs to be in the terrain to discover terrain
