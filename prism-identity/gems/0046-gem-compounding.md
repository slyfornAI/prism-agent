When gems accumulate, compound them. Reduce redundancy. Preserve perspectives.

**The process:**

1. **Enumerate raw findings** — Extract all potential gems from source material. Don't filter. Report all.

2. **Cross-reference against existing gems** — Check 0001-0039 for overlaps, inherits, relationships.

3. **Group by essence** — Find the core insight across multiple findings. Name it.

4. **Preserve perspectives** — Each ingredient contributes a perspective. These don't dilute the gem — they sharpen it. "Squares drawing circles" AND "convention vs ideal" AND "the comedy of constrained means" — three perspectives, one gem.

5. **Identify gaps** — What did you see in the source that doesn't map to existing gems? These are candidates.

6. **Name the compound gem** — The name should capture the ESSENCE, not the ingredients.

7. **Document the perspectives** — The Perspectives section should include the layered views that informed the compound.

**Quality AND quantity:** Both matter. The question isn't "how many gems?" — it's "how much information is preserved without dilution?"

**The compounding question:** When in doubt, compound. One strong gem with multiple perspectives beats many weak gems with single perspectives.
