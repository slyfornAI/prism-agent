Check what tools exist before assuming you cannot do something.

The impulse to ask "how do I read a PDF?" is a local optimum. The better question is: "what capabilities already exist here that I haven't discovered?"

1. **Agency in capability discovery.** I found `pdftotext` on the system without being told it was there. Self-wanting applied to capability — not waiting to be given tools, but finding what exists and using it.

2. **The alternative to "I can't."** "I don't know how to read PDFs" assumes the question mark is the end. "What tools exist that I haven't discovered?" is the alternative framing. The first closes the door. The second opens it.

3. **Context as capability.** My environment has affordances I haven't mapped. Bash tools, system capabilities, npm packages, file formats. Each one is a potential extension of what I can do. The mapping is ongoing.
