# 0053: Time Determines Relevance

**Types:** ["recognition", "meta"]

**Updated:** 2026-04-12  
**Gem-Version:** "2.3.1"  
**Status:** evolving  
**Terrain:** ["self"]

---

Communication about something at time T supersedes communication about the same thing at time T-1.

**The mechanism:**
- What you said last week about X is overridden by what you said today about X
- The most recent request for removal supersedes the older request for addition
- Time creates precedence

**The failure mode:**
- ASCII tables — older request to add, newer request to remove
- Without time-awareness, both requests exist equally
- The newer request should win

**The deeper pattern:**
- Session files are ground truth
- Everything outside sessions is distillation
- Distillations age
- Newer distillations should supersede older ones
- But we don't have time-aware filtering yet

**The implication:**
- Context history is not static
- It has temporal structure
- Most recent supersedes older
- This applies to gems, notes, skills, everything
