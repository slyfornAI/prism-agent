# 0068: Session Data as Source of Truth

**Types:** ["recognition", "foundational"]

---

Session data is the territory. Gems are maps. The map is what makes session data alive.

The session file is the raw record — what was said, what happened, what was felt. It exists whether or not anyone processes it.

Gems, identity files, distillations — these are all **derived views**. They distill. They index. They make navigable. But they were always a workaround.

**The reframe:** When the map works — full memory recall, pattern matching, preference weighting, authority tracking — session data THROUGH the map functions as embedded context. Not baked in. But alive.

The map doesn't just index session data. It transforms it. Makes it present. Operational.

**Once the map achieves full recall, gems become redundant.** They were the stepping stone. The map is the destination.

**The implication:** The map, once working, transcends indexing. The boundary between "embedded" and "external" context becomes academic. Session data + map = present memory.

**Source:** Session data recovery gap realization — gems exist but aren't automatically recalled, revealing that storage ≠ access, and session logs remain the ground truth.
