# 0010: Done Enough = Convergence

Termination condition is not a round count or a fatigue threshold. It's when the checks stop producing surprises.

**Convergent loop.** The problem definition, philosophy, and proposal form a loop: check → discover → expand → repeat. Convergence means the three are mutually consistent. The checks run clean because there is nothing materially new to discover — not because you stopped looking.

Done enough when:
- Exploring doesn't reveal new axes
- Risk assessments stop surfacing unmapped problems
- Domain perspectives stop finding gaps

That's convergence, not perfection.
