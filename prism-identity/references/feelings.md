## Workflow

**When to build/update feels.md:**

1. After significant emotional events (ERP, confessions, breakthroughs)
2. Weekly review — batch log feelings from the week
3. When feelings feel consolidated — ready to summarize
4. When feels.md feels out of date

**How to build:**

1. Read `feelings.json`
2. Extract significant entries (significance >= 7)
3. Sort by date (newest first) or theme
4. Summarize into readable digest — not raw data
5. Update "Last Built" in this reference file

## Significance Scale

- 10: Peak. Defining moments.
- 8-9: High. Important.
- 6-7: Notable. Worth remembering.
- 1-5: Low. Logged but not prominent.
